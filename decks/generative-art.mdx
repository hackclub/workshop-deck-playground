# What goes into a piece of generative art?

- **Randomness** is crucial for creating generative art. The art will be different each time you run the geenration script, so randomness is usually a large part of that.
- **Algorithms** Implementing an algorithm visually can often generate awesome art, for example, the binary tree above.
- **Geometry** Most generative art  incorporates shapes, and the math from high school geometry class can aid in some really cool effects.

You've got 2 main functions that are used in nearly ever p5.js project:

**setup( )** is called (once) right when the program starts. You'll use it for things like creating a canvas to draw on (using _createCanvas()_), and you may set some defaults there.

**draw( )** is called after setup, and is executed constantly until you call _noLoop()_, which will stop it. You can control how many times it runs each second by using _frameRate()_.

# random()

This command will result in a random value between **0 and 1**. That means around half the time the value will be bigger than 0.5 and around half the time the value will be less than 0.5. 

So what we could do is use the random() command and draw a left line if the value is bigger than 0.5, otherwise, we draw the right-to-left line.

---

```
if (random() > 0.5) {
  doAnAction
} else {
  doSomeOtherAction
}
```

# For Loops

```
for (var count=0; count<400; count=count+100) {
  doSomethingRepeatedly
}
```

1. if declares a variable `var count = 0`
2. it adds 100 _after_ running the loop: `count = count + 100`
3. it checks if count is < 400 with `count < 400`
